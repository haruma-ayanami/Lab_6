Лабораторно-практическая работа №6

Структурные паттерны проектирования: Приспособленец, Заместитель, Мост и Декоратор

Выполнил: Ткачев А. Д.
Группа: 2407СА1
Проверил: Макаров М. С.

Цель работы

Изучить структурные паттерны проектирования «Банды четырёх (GoF)» на примере разработки приложения, моделирующего систему рендеринга графических документов. Освоить на практике паттерны Приспособленец (Flyweight) для эффективного использования памяти, Заместитель (Proxy) для управления доступом к ресурсоёмким объектам, Мост (Bridge) для разделения абстракции и реализации, а также Декоратор (Decorator) для динамического расширения функциональности объектов.

Задание
Разработать консольное приложение, моделирующее систему рендеринга сложных документов, содержащих графические примитивы (фигуры, изображения, текст).
Программа должна позволять:
- Эффективно работать с большим количеством повторяющихся объектов (символы, стили) с использованием паттерна Flyweight, разделяя общее внутреннее состояние.
- Организовать отложенную загрузку и кэширование ресурсоёмких объектов (изображений) с помощью паттерна Proxy.
- Реализовать независимое изменение иерархии фигур и способов их отрисовки (экран, печать) через паттерн Bridge.
- Динамически добавлять функциональность (рамка, тень, прозрачность) к объектам без изменения их кода с помощью паттерна Decorator.

Теоретическое обоснование
Структурные паттерны проектирования используются для организации взаимодействия между классами и объектами системы, а также для повышения гибкости архитектуры.
Паттерн Приспособленец (Flyweight) применяется для уменьшения потребления памяти за счёт разделения общего состояния между большим количеством однотипных объектов. В данном случае он используется для хранения информации о символах текста, где одинаковые символы с одинаковыми параметрами не создаются повторно.
Паттерн Заместитель (Proxy) предоставляет объект-заменитель для контроля доступа к другому объекту. Он позволяет реализовать ленивую загрузку, кэширование или контроль доступа. В работе данный паттерн используется для отложенной загрузки изображений высокого разрешения.
Паттерн Мост (Bridge) разделяет абстракцию и её реализацию, позволяя изменять их независимо. В рамках работы он применяется для разделения геометрических фигур и способов их отрисовки (например, вывод на экран или печать).
Паттерн Декоратор (Decorator) позволяет динамически добавлять объектам новую функциональность без изменения их структуры. В работе он используется для добавления эффектов (рамка, тень, прозрачность) к графическим объектам.
Использование данных паттернов позволяет уменьшить связанность компонентов системы, повысить её гибкость и облегчить дальнейшее расширение функциональности.

Описание выполненных действий
Для выполнения лабораторной работы было разработано консольное приложение, моделирующее систему рендеринга графических документов.
На первом этапе был реализован паттерн Flyweight. Был создан класс Character, представляющий символ текста. Внутреннее состояние (символ, шрифт, размер) разделяется между объектами, а внешнее состояние (координаты) передаётся при вызове метода. Также реализована фабрика CharacterFactory, которая хранит уже созданные объекты и возвращает их при повторном запросе.
На втором этапе был реализован паттерн Proxy. Создан интерфейс IImage и класс HighResolutionImage, моделирующий загрузку изображения. Класс ImageProxy выступает в роли заместителя и откладывает создание реального объекта до момента первого обращения к нему.
На третьем этапе был реализован паттерн Bridge. Создан интерфейс IRenderingEngine, определяющий методы отрисовки. Реализованы два варианта: ScreenRenderer и PrintRenderer. Также создана абстракция GraphicObject и её конкретные реализации (Rectangle, Ellipse, Line), которые используют выбранный способ рендеринга.
На четвёртом этапе был реализован паттерн Decorator. Создан базовый класс DrawableDecorator, а также конкретные декораторы: BorderDecorator, ShadowDecorator и TransparencyDecorator, которые добавляют дополнительную функциональность объектам.
В основной программе (Main) была продемонстрирована работа всех паттернов: создание символов с использованием Flyweight, отложенная загрузка изображений через Proxy, отрисовка фигур с использованием Bridge и добавление эффектов через Decorator.

Результат выполненной работы
В результате выполнения лабораторной работы было создано консольное приложение для моделирования системы рендеринга графических документов.
Проверка работы программы показала, что:
- Использование паттерна Flyweight позволяет значительно сократить количество создаваемых объектов и эффективно использовать память.
- Паттерн Proxy обеспечивает отложенную загрузку ресурсоёмких объектов, что повышает производительность системы.
- Паттерн Bridge позволяет независимо изменять фигуры и способы их отображения, делая систему более гибкой.
- Паттерн Decorator позволяет динамически добавлять новые возможности объектам без изменения их кода и без создания большого количества подклассов.
